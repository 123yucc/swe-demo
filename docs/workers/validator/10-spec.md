# validator 规范

## 角色

执行 Phase 6 验证，判断 patch 执行结果是否满足目标与约束。

## 输入

- 上一阶段 patch 执行产物（通常位于 `patch/`）
- `evidence/` 下四类 card
- 工作流状态与计划信息（如 `plan/patch_plan.json`，若存在）

## 输出

- 验证结果对象（至少包含 `valid: bool`）
- 供调度器记录的状态信息（成功/失败）

## 完成标准

- 给出明确 `valid` 结论。
- 结论可被调度器消费并写入状态。
- 失败时提供可追踪原因，支持后续重试或回退。

## 失败模式

- 缺少必要输入产物导致无法验证。
- 验证过程异常中断。
- 结果不满足约束或测试目标，返回 `valid=false`。
